## 1. 제품 개요 및 목표

### 목적

카카오톡, 전화, 엑셀로 분산된 업체 협업 업무를 하나의 서비스에서 통합 관리하여 업무 누락과 일정 혼선을 줄인다.

### 타겟 사용자

외부 업체 여러 곳과 동시에 프로젝트를 진행하는 소규모 사업자 또는 내부 운영팀 담당자.

### 핵심 가치 제안

"엑셀보다 쉽고, 카톡보다 안 잊어버리는 협업 서비스"

| 문제 상황 | 현재 방식 | 제공 가치 |
|-----------|-----------|-----------|
| 업무 누락 | 카카오톡 메시지 흘러내림 | 업무 상태판에서 항상 확인 가능 |
| 일정 혼선 | 개인 캘린더, 메모 분산 | 캘린더 화면에서 업체별 일정 통합 확인 |
| 담당자 혼동 | 구두 전달 | 업무마다 담당자 명시 |
| 진행 상태 불투명 | 전화/문자로 확인 | 칸반 보드 상태 실시간 반영 |

## 2. 사용자 역할 정의

MVP에서는 **자유 가입** 방식을 채택한다. 1~3인의 신뢰된 내부 팀원이 각자 이메일로 가입하면 동일한 워크스페이스에 자동으로 접근된다. 별도 초대 절차는 없다.

### 2-1. MVP: 단일 역할 (Admin)

- 가입한 모든 사용자는 동등한 권한을 가짐
- 업체 등록/수정/삭제 가능
- 업무 생성/수정/삭제/상태변경 가능
- 댓글 작성 가능

> profiles.role 컬럼은 기본값 `admin`으로 유지한다. 2차 기능에서 멤버 권한 분기가 필요할 경우 이 컬럼을 활용한다.

> MVP 범위 메모: 외부 업체 담당자 계정, 팀원 초대 이메일, 권한 분리는 2차로 연기한다. MVP에서 업체 담당자는 시스템 내 데이터로만 관리되며 로그인 주체가 아니다.

## 3. 사용자 여정

### 3-1. 신규 관리자 온보딩 여정

```
랜딩 페이지(/)
  ↓ [회원가입 버튼 클릭]
회원가입 페이지
  ↓ [이메일/비밀번호 입력 후 가입]
이메일 인증 안내 페이지
  ↓ [이메일 링크 클릭]
대시보드 페이지 (첫 진입 → 빈 상태 안내 표시)
  ↓ [업체 등록 CTA 클릭]
업체 등록 모달
  ↓ [업체 저장 완료]
업체 목록 페이지
  ↓ [업무 추가 버튼 클릭]
업무 생성 모달
  ↓ [업무 저장 완료]
업무 관리 페이지 (칸반 보드)
```

### 3-2. 기존 사용자 일상 업무 여정

```
로그인 페이지
  ↓ [로그인 성공]
대시보드 페이지
  ↓ [오늘 마감 업무 카드 클릭]
업무 상세 모달
  ↓ [상태 변경 또는 댓글 작성]
상태 변경 완료 → 업무 관리 페이지 칸반 보드 반영
```

### 3-3. 업무 상태 관리 여정

```
업무 관리 페이지 (칸반 보드)
  ↓ [카드 드래그 또는 상태 드롭다운 변경]
  대기 → 진행중 → 확인요청 → 완료
  ↓ [확인요청 상태 시 댓글로 내용 공유]
업무 상세 모달 (댓글 작성)
  ↓ [저장 완료]
대시보드의 최근 댓글 목록에 반영
```

## 4. 기능 명세 (MVP 중심)

### 4-1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 업체 등록/조회/수정/삭제 | 협업 업체 정보를 CRUD로 관리 | 업무와 연결되는 핵심 기준 데이터 | 업체 목록 페이지, 업체 상세 모달 |
| **F002** | 업무 생성/조회/수정/삭제 | 업체별 업무를 생성하고 담당자·일정·상태를 관리 | 서비스의 핵심 가치인 업무 추적 | 업무 관리 페이지, 업무 상세 모달 |
| **F003** | 업무 상태 변경 (칸반) | 대기→진행중→확인요청→완료 상태를 드래그 또는 드롭다운으로 변경 | 진행 현황 가시화가 핵심 기능 | 업무 관리 페이지 |
| **F004** | 대시보드 요약 | 오늘 마감 업무, 진행중 업무 수, 완료 업무 수, 최근 댓글 표시 | 서비스 진입 시 업무 현황 즉시 파악 | 대시보드 페이지 |
| **F005** | 업무별 댓글 | 업무 상세에서 댓글 작성/조회 | 카카오톡 대신 업무 컨텍스트 내 커뮤니케이션 | 업무 상세 모달 |
| **F006** | 업무 캘린더 | 업무 마감일을 월간 캘린더로 시각화, 업체별 필터 | 일정 혼선 해소 | 캘린더 페이지 |
| **F007** | 업무 우선순위 설정 | 높음/중간/낮음 우선순위 지정 및 필터링 | 업무 선별 집중도 향상 | 업무 관리 페이지, 업무 상세 모달 |
| **F008** | 카카오 알림톡 — 일일 요약 | 매일 오전 9시, 마감일이 오늘인 업무를 업체 담당자 번호로 자동 발송 | 업무 누락 방지, 외부 업체 능동적 알림 | 자동 실행 (Vercel Cron) |
| **F009** | 카카오 알림톡 — 업무 등록 즉시 발송 | 업무 생성 시 선택적으로 관련 업체에 즉시 알림 발송 | 업무 등록 즉시 업체 공유 | 업무 생성 모달 |

### 4-2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | 기본 인증 | 이메일/비밀번호 회원가입, 로그인, 로그아웃, 이메일 인증 | 서비스 이용을 위한 최소 인증 | 로그인 페이지, 회원가입 페이지 |
| **F011** | 사용자 프로필 최소 관리 | 표시 이름 저장 (업무 담당자 표시에 사용) | 담당자 이름 표시를 위한 필수 데이터 | 회원가입 페이지 |
| **F012** | 업체별 업무 필터 | 업무 목록을 업체 기준으로 필터링 | 업체 단위 업무 파악을 위한 필수 기능 | 업무 관리 페이지 |

### 4-3. MVP 이후 기능 (제외)

- 파일 첨부 기능 (Supabase Storage 연동)
- 업체별 외부 사용자 계정 발급 및 권한 관리
- 공유 링크로 외부 공개
- 자동 반복 업무 설정
- 견적/발주 관리
- AI 일정 추천
- 업무 통계 분석 차트
- 모바일 전용 앱 (PWA 포함)

## 5. 메뉴 구조

```
TATACO 내비게이션
├── 대시보드                        기능: F004 (업무 현황 요약)
├── 업무 관리                       기능: F002, F003, F007, F012 (칸반 보드, 상태관리, 우선순위, 필터)
├── 업체 관리                       기능: F001 (업체 CRUD)
├── 캘린더                          기능: F006 (월간 일정 시각화)
└── 인증 (비로그인 시)
    ├── 로그인                      기능: F010
    └── 회원가입                    기능: F010, F011

사용자 메뉴 (로그인 후 헤더 우측)
├── 내 정보                         기능: F011 (표시 이름 확인)
└── 로그아웃                        기능: F010
```

### 대시보드 페이지

> **구현 기능:** F004, F005 | **메뉴 위치:** 좌측 사이드바 최상단

| 항목 | 내용 |
|------|------|
| **역할** | 서비스 진입 시 업무 전체 현황을 한눈에 파악하는 홈 화면 |
| **진입 경로** | 로그인 성공 후 자동 리디렉션, 사이드바 '대시보드' 클릭 |
| **사용자 행동** | 오늘 마감 업무 확인, 진행중 업무 수 확인, 최근 댓글 클릭으로 해당 업무 이동 |
| **주요 기능** | - 오늘 마감 업무 카드 목록 (마감일 = 오늘인 업무)<br>- 진행중 업무 수 / 완료 업무 수 요약 배지<br>- 최근 댓글 5건 (작성자, 업무명, 내용 미리보기)<br>- 빈 상태(업무 없음) 시 업체 등록 유도 CTA 표시 |
| **다음 이동** | 업무 카드 클릭 → 업무 상세 모달, 업체 등록 CTA → 업체 목록 페이지 |

### 업무 관리 페이지

> **구현 기능:** F002, F003, F007, F012 | **메뉴 위치:** 좌측 사이드바 '업무 관리'

| 항목 | 내용 |
|------|------|
| **역할** | 모든 업무를 칸반 보드 또는 리스트 형태로 조회하고 상태를 관리하는 핵심 화면 |
| **진입 경로** | 사이드바 '업무 관리' 클릭, 대시보드 업무 카드 클릭 후 모달 닫기 |
| **사용자 행동** | 업무 카드 드래그로 상태 변경, 업체 필터로 특정 업체 업무만 조회, 업무 카드 클릭으로 상세 열기, 우선순위 필터로 정렬 |
| **주요 기능** | - 칸반 4컬럼: 대기 / 진행중 / 확인요청 / 완료<br>- 업무 카드: 업무명, 담당자, 마감일, 우선순위 뱃지, 업체명 표시<br>- 업체 필터 드롭다운 (전체 / 개별 업체 선택)<br>- 우선순위 필터 (전체 / 높음 / 중간 / 낮음)<br>- 업무 추가 버튼 → 업무 생성 모달 오픈<br>- 드래그 앤 드롭으로 상태 컬럼 간 이동 |
| **다음 이동** | 업무 카드 클릭 → 업무 상세 모달, 업무 추가 → 업무 생성 모달 |

| **주요 기능** | - 업무명, 관련 업체, 담당자, 시작일, 마감일 수정 폼<br>- 상태 드롭다운 (대기/진행중/확인요청/완료)<br>- 우선순위 드롭다운 (높음/중간/낮음)<br>- 메모 텍스트 영역<br>- 댓글 목록 (작성자, 내용, 작성 시각 표시)<br>- 댓글 입력창 및 등록 버튼<br>- 업무 삭제 버튼 (관리자 전용, 확인 다이얼로그 포함) |
| **다음 이동** | 저장 → 모달 닫기 후 업무 관리 페이지 반영, 삭제 → 업무 목록에서 제거 |

### 업체 상세 모달

> **구현 기능:** F001 | **메뉴 위치:** 업체 카드 클릭 시 오버레이

| 항목 | 내용 |
|------|------|
| **역할** | 단일 업체의 상세 정보를 확인·수정하고 해당 업체의 연결 업무를 조회하는 화면 |

## 9. 인증 및 권한 체계

### 9-1. Supabase RLS 정책 설계

MVP는 자유 가입 + 단일 역할 구조이므로 **"로그인 = 전체 접근"** 원칙을 적용한다.

```sql
-- profiles:  로그인 사용자 전체 조회 가능 / 본인만 수정 가능
-- companies: 로그인 사용자 CRUD 전체 가능 (auth.uid() IS NOT NULL)
-- tasks:     로그인 사용자 CRUD 전체 가능 (auth.uid() IS NOT NULL)
-- comments:  로그인 사용자 전체 조회/작성 가능 / 본인 댓글만 삭제 가능
```

> 2차 기능에서 멤버 권한을 분리할 때 profiles.role = 'admin' 조건을 추가하면 된다.

### 9-2. 라우트 보호 규칙

| 경로 | 접근 조건 |
|------|----------|
| `/` | 공개 (랜딩 페이지) |
| `/auth/login` | 비로그인 사용자 |
| `/auth/sign-up` | 비로그인 사용자 |
| `/dashboard` 이하 전체 | 로그인 필수, 미인증 시 `/auth/login`으로 리디렉션 |

### 9-3. 초기 역할 할당

- 회원가입 시 Supabase DB 트리거로 `profiles` 레코드 자동 생성
- 기본 역할은 `admin`으로 설정 (MVP 전원 동등 권한)
- 별도 초대 절차 없음 — 이메일/비밀번호로 직접 가입하면 즉시 이용 가능
- 2차에서 멤버 초대 기능 추가 시 초대된 사용자를 `member`로 지정
